# Versioning Strategy

This project adheres to [Semantic Versioning](https://semver.org/) (SemVer) standard: `MAJOR.MINOR.PATCH`.

## Version Increments

- **MAJOR** version (e.g., `1.0.0` -> `2.0.0`):
  - Incremented when incompatible API changes are made.
  - Indicates significant architectural shifts, complete UI overhauls, or breaking changes in external APIs.
- **MINOR** version (e.g., `1.1.0` -> `1.2.0`):
  - Incremented when new functionality is added in a backwards-compatible manner.
  - Indicates new features, endpoints, or significant non-breaking enhancements.
- **PATCH** version (e.g., `1.1.2` -> `1.1.3`):
  - Incremented when backwards-compatible bug fixes are made.
  - Indicates internal fixes, security patches, or performance improvements that do not affect the public API.
